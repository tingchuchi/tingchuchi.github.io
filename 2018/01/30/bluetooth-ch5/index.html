5.藍牙技術-HCI Layer介紹

Host Controller Interface Layer(HCI) 是Bluetooth 設計用來提供 Host 與 Controller 溝通的介面，目前定義了Transport可使 Host 與 BT module 能實際連結，目前已經制定的 Transport 有UART(H4)、USB、SPI、H3/RS232。

![](/images/bt_11.png)

HCI 有四種基本型態的封包，在不同的傳輸層透過不同的方式區別

Commands -用於Host 對 BT Module的控制

A. UART Header: 0x01
B. USB: control endpoint (endpoint 0)

ACL –常用於傳輸一般性資料，具有基本的資料可靠性保證

A. UART Header: 0x02
B. USB: Bulk endpoint

SCO - 用於語音資料的傳遞

A. UART Header: 0x03
B. USB: Isochronous endpoint

Event- 用於BT Module 回應 Host 的控制

A. UART Header: 0x04
B. USB: Interrupt endpoint

詳細的Command 可以參考Core.v5.0.pdf 5-7章節(HCI Data Formats、HCI Configuration Parameters、HCI Commands and Events)
