3. 低功耗藍牙規範-架構

發表於 2018-11-30 | 分類於 藍牙 | 閱讀次數:

低功耗藍牙(BLE)的架構如下圖所示：分為三個部分，Controller、Host、Application。



圖1. 低功耗藍牙架構圖

Controller通常為一個實體設備，它能夠發送及接收無線電訊號，並懂得如何將這些訊號解讀成能攜帶訊息的Data packet。

Host是軟體堆疊(Software stack)，用來管理兩台或多台裝置間如何通訊以及如何利用無線電同時提供數種不同的服務。

Application使用Software stack，進而利用Controller來實現現實中應用。

Controller內有Physical Layer及Link Layer，又有Direct Test Mode和HCI Layer的下半部分。在Host內有三個協定：L2CAP、Attribute Protocol、Security Manager Protocol，此外還包含GATT、GAP等。在本章節，根據低功耗藍牙架構的每個部份做簡要介紹，在後續章節會針對每個部份做詳細說明。

藍牙 5.0後及未來發展趨勢

前言

藍牙(Bluetooth) 從 1998 年開始發展至今，版本也從1.0 發展成熟至現今的 5.0。以消費者來說，關注的當然不是新一代的標準，而是未來在應用面上會有什麼樣的發展。
在本篇文章中，針對 BLE mesh , Ture Wireless, BLE Audio筆者認為可能發展的三個面向來探討。

BLE mesh (網狀網路)

大部分藍牙低功耗設備(BLE devive)都為一對一通訊，在規格中稱為”微微網(Piconet)。相比之下，mesh網路則具有多對多通訊架構。
在藍牙 5.0 之前，藍牙 IC製造商各自定義了mesh 標準。其原理是利用 BLE 廣播封包來進行廣播資料。 其中每個BLE mesh節點都可以當作廣播(中介)的角色，因BLE相對於WIFI的耗電及成本低，所以對於整個mesh拓樸相當容易，在智慧家居的應用上提供了另一個選擇。
但早期的BLE mesh 並不是沒有缺點的，在2016年前，由於BLE 版本(<5.0)廣播資料支援最大單元傳輸量(PDU)為 31 byte，能夠傳遞的資料少；其次，因藍牙應用開發商採用的mesh通訊協定並不統一，常常會有干擾以及傳輸延遲的問題；另外，安全性上也因為廣播資料大小的限制，所以能做的也有限，這也成為未來商品化的隱憂。
於是在藍牙5.0，SIG開始規範BLE mesh協定，並且大幅了提升了通訊距離及廣播資料傳輸量(X8)。隨著BLE mesh 協定標準化，BLE mesh在IOT的應用上也將更為成熟。

Ture Wireless (TWS) 真無線耳機

Ture Wireless並不是新一代通訊協定，早在BT 2.0 時，就有出現這樣的應用。這邊要說明的是，目前市面上藍牙音訊(喇叭、耳機…)應用還是使用傳統藍牙(BR/EDR)，而非4.0之後推出的BLE。所以市面上標榜的新一代的4.0/4.1/4.2/5.0藍牙晶片造成音質升級都只是噱頭(或許藍牙製造商也不清楚之間的差異)。但傳統藍牙的版本提升到4.0，對於功耗上卻是有了明顯的改善。
也因為功耗的改善，從早期TWS只能做在喇叭上(因需電池體積大)，開始慢慢地移植到耳機。
TWS的架構為手機(Sorce)將音樂傳輸給其中一隻耳機，再由其中一隻耳機傳音樂到另一隻耳機。這代表著，中間有一隻耳機同時要接收來自手機(Sorce)的音訊，又要發送給另一隻耳機。在有限的頻寬下需要做到這件事情，所以可能會造成

  1. 因為晶片在耳機上，傳輸音訊編解碼總是造成CPU的繁忙，耗電太大。
  2. 傳輸的品質不好。(因頻寬有限，再加上戴在人的雙耳，人身體構造大部分為水，在2.4G Hz頻段中容易受干擾) 常常會在副耳聽到斷線的現象。
    這些缺點當然在耳機製造商也看到了，所以有了AirPods +充電盒的問世。
    其架構主要在兩隻耳機間，用了另外的通訊協定(非藍牙)來傳輸，來提升傳輸的品質，當然相對應的製造成本也提升不少。
    事實上各家IC製造商也開始針對Ture Wireless (TWS)，提出更好的解決方法。譬如開發兩耳間特殊的音訊編解碼、或是優化藍牙的底層、減少功耗等等。期待在未來 ，TWS能夠真正取代一般藍牙耳機。

BLE Audio

在藍牙4.0開始有了BLE的制定，其特色為連接速度快速，並且相比傳統藍牙(BR/EDR)省電的許多，但相對地傳輸的速度則沒辦法進行音訊的應用。
藍牙5.0開始提升了BLE的傳輸速率到2Mb/S。這樣的速率已經可以進行 Audio 的應用了。而事實上在新的穿戴裝置或是遙控器上，各家開發商也慢慢地將聲控功能加入BLE裝置中。
SIG也把BLE Audio 列為未來藍牙和新協定版本的計畫中。在未來，我們不需要再多使用一顆傳統藍牙晶片就可以應用在音樂上了。這將大大減少藍牙晶片的體積、耗電。

Link Layer 包含了 Logic Transport 及 Logic Link

Logic Link

在藍牙已連接的設備之間，根據Physical Link建立Logic Link，我們可以想像成將一條道路作為physical link，這條路根據行車的用途可以劃分為許多logic link (例如:直行、右轉、左轉車道..)。Logic Link可分為三類

A. 用來管理底層(Physical Link)的控制類: AMP-C、ACL-C、PSB-C、LE-C、ADVB-C
B. 傳輸用戶資料類: AMP-U、ACL-U、PSB-U、LE-U、ADVB-U
C. 其它特殊類型: 串流(stream)、PBD(Profile Broadcast Data)
